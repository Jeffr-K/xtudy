# Preface

객체 간 관계를 다루는 패턴이 구조 패턴(structural pattern) 이다.

먼저 클래스 사이의 복잡한 계층 관계를 만들지 않고도 객체의 기능을 확장하는 법을 알아본다. 그리고 미래의 변경 사항을 반영하며 과거의 설계를 수정하는 방법을 살펴본다.

메모리 사용량을 줄이는 방법도 다루어본다.
